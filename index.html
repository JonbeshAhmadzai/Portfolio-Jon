Case Study 1 Belgian Real Estate Price Prediction Python · CatBoost · XGBoost · LightGBM · Geopy · Scikit-learn · FastAPI · Docker · Render

The Problem

Belgian property prices vary enormously depending on size, type, and location. Raw zip codes alone do not explain enough, so the challenge was building a useful model from publicly available listing data.

The Approach

A multi-threaded pipeline collected 23,000+ Immovlan listings across Belgian provinces. Standard property features were enriched with geographic context: zip codes were geocoded, distances to Brussels, Antwerp, and Ghent were computed, and features like area per room, outdoor space ratio, land usage ratio, and nearest-city distance were engineered.

The Tradeoffs

Linear Regression was too simple for the pricing patterns. Random Forest reached 0.968 training R² but dropped to 0.768 on test, showing overfitting. CatBoost found the best balance with 0.933 training R², 0.786 test R², and a MAE of €74,252.

The Result

CatBoost was selected, serialized with joblib, and structured for API deployment. Living area, nearest-city distance, latitude, and longitude were key predictors, proving that geographic intelligence improved predictions beyond property attributes alone.

Case Study 2 Satellite Crop Classification Python · Sentinel-2 · Copernicus STAC API · Azure Blob Storage · GeoPandas · Rasterio · Scikit-learn

The Problem

Crop mapping at regional scale matters for agricultural planning, environmental monitoring, and land-use decisions, but manual classification does not scale. The goal was to classify crop types directly from satellite imagery.

The Approach

The pipeline queries Copernicus STAC for Sentinel-2 L2A scenes, filters by cloud cover, downloads selected bands, uploads them to Azure Blob Storage, applies cloud masking, and builds monthly median composites. A 117-feature stack combines spectral bands with NDVI, NDWI, and NDBI. LPIS/Walous parcel polygons are rasterized as training labels for a Random Forest classifier.

The Tradeoffs

Monthly compositing was chosen over single-scene analysis so cloudy scenes would not break the pipeline. Majority-filter post-processing removes isolated noisy pixels while slightly softening boundaries, which was acceptable for cleaner crop maps.

The Result

The pipeline exports reclassified GeoTIFF crop maps. Feature importance showed that seasonal changes in spectral indices were more predictive than any single band, validating the time-series feature stack.

Case Study 3 HCMST Relationship Survival Analysis Python · Pandas · Scikit-learn · SciPy · K-Means · Logistic Regression · pyreadstat · Plotly

The Problem

Does meeting online predict whether a couple stays together long term, or do other factors matter more? The question is both research-oriented and practical for platforms trying to understand durable relationships.

The Approach

The HCMST 2017-2022 longitudinal survey contains 3,510 respondents, 725 variables, and three waves. After filtering and cleaning, clustering and survival modelling ran in parallel. K-Means without gender produced six interpretable profiles, while logistic regression tested four predictor sets on respondents with known Wave 3 outcomes.

The Tradeoffs

Because 89% of relationships survived, accuracy would have been misleading. A model predicting “survived” for everyone would look strong without learning anything. ROC AUC was used instead, with the best model reaching 0.678 versus 0.631 for meeting-context-only predictors.

The Result

Meeting online had some predictive value, but relationship quality and structure were stronger signals. For a dating platform, the implication is clear: product design should prioritize compatibility and relationship quality, not only discovery.

Technical Notes

Decisions Behind the Work

Why I removed gender from my clustering model

With gender included, K-Means split clusters almost entirely along gender lines. Removing it forced the model to find structure in age, relationship duration, quality, marital status, and meeting context, producing six more meaningful profiles.

Why accuracy was the wrong metric

Because 89% of relationships survived to Wave 3, a model predicting “survived” for everyone would score 89% accuracy. ROC AUC measured class separation instead, revealing a real improvement to 0.678.

How geographic features improved price prediction

Adding latitude, longitude, and distances to Brussels, Antwerp, and Ghent helped CatBoost capture regional pricing patterns. Nearest-city distance became one of the strongest predictors.

Why monthly Sentinel-2 compositing matters

Single scenes over Belgium are often cloud covered. Monthly median compositing creates cleaner land-surface representations and captures seasonal crop changes across the growing season.
